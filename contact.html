Questions, bug reports, and feature ideas are welcome. The fastest way to reach the project is through GitHub issues so discussion stays visible and actionable.

Support channels

  • General questions: open a GitHub issue with context and expected behavior.
  • Bug reports: include SQL sample, generated LINQ, and what output was expected.
  • Feature requests: describe the query pattern and business scenario.

Create a new issue on GitHub

How to report conversion issues well

High-quality reports reduce turnaround time and improve release planning accuracy. Please include:

  • SQL input that reproduces the issue.
  • Generated LINQ output from SQLinq.
  • Expected LINQ or expected behavior.
  • Whether the issue is parser, conversion logic, or formatting related.
  • Any database-specific assumptions in the SQL.

Collaboration expectations

SQLinq is built in the open with transparent milestones. Polite, specific, and reproducible issue reports help maintain a high-quality conversion roadmap for all users.
