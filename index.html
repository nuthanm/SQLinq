SQL input
WITH Ranked AS (
  SELECT *, ROW_NUMBER() OVER (
    PARTITION BY CustomerId ORDER BY OrderDate DESC
  ) AS rn
  FROM Orders
)
SELECT * FROM Ranked WHERE rn = 1;
LINQ preview
orders
  .GroupBy(o => o.CustomerId)
  .Select(g => g
    .OrderByDescending(o => o.OrderDate)
    .First());
Estimated execution plan
  • Clustered Index Scan 62% Orders
  • Sort 18% OrderDate DESC
  • Segment / Sequence Project 12% ROW_NUMBER
  • Filter 8% rn = 1
Suggestions

Index opportunity

Scan on Orders dominates cost. Consider covering index on (CustomerId, OrderDate DESC).

LINQ shape note

GroupBy + First may translate differently under EF Core. Prefer ordered take if provider support is limited.

Semantic parity

Window function rewritten to per-group ordering. Verify NULL handling for OrderDate.

Conversion concepts (hierarchy)
    • WithClause → CTE Ranked
    • Select · Star expansion deferred
    • OverClause · ROW_NUMBER
    • Orders → IQueryable<Order>
    • CustomerId : int · OrderDate : datetime2
    • Window(partition, order) → GroupBy + ordered First
    • Filter(rn = 1) → folded into First()
    • Rule: WindowToGroupHead
    • Expression tree → Roslyn syntax
    • Formatting · indent 2 · fluent chain
