Animated workflow: SQL to LINQ conversion

Same conversion model across releases: parse SQL, map intent to domain symbols, and emit final LINQ query.

1. Parse

Read SQL tokens and build query structure.

  • C#/.NET concepts: tokenization, AST model, pattern matching
  • Typical APIs: Span, Regex, immutable syntax nodes

2. Map

Bind SQL names to model/entity members and semantics.

  • C#/.NET concepts: symbol tables, dictionaries, type metadata
  • Typical APIs: Dictionary, reflection metadata, Roslyn symbols

3. Final query

Generate equivalent LINQ in method or query syntax.

  • C#/.NET concepts: expression trees, IQueryable pipeline
  • Typical APIs: Enumerable/Queryable operators, lambda expressions

Developer guide

Start converting in under five minutes

Phase 1 is VS Code first — select SQL, convert, inspect plan, accept LINQ. Editor-specific polish and other hosts come later; the core flow stays shared.

VS Code

Supported now

Install from the VS Code Marketplace, open a SQL file, and launch the conversion popup from the command palette or keyboard shortcut.

  • Best for lightweight SQL to LINQ conversion demos.
  • Uses the same shared conversion core as Visual Studio.
  • Works with the free, distraction-free extension flow.

Visual Studio

Not in Phase 1

Visual Studio stays in the roadmap, but Phase 1 is being kept narrow so the VS Code experience can ship first.

  • Later host integration, not part of the first public release.
  • Will reuse the same conversion core and plan language.
  • Shown here to make the supported scope explicit to visitors.

JetBrains

Not in Phase 1

JetBrains support is intentionally deferred. The shared converter core is being designed so a future plugin can join without changing the rewrite engine.

  • Not supported in the current release target.
  • Tracked as future host integration work.
  • Keep the shared core extensible before adding new IDE shells.

Phase 1 ships for VS Code only. Visual Studio and JetBrains are shown as future host targets so visitors can see the roadmap clearly.
