About SQLinq

SQLinq exists to help developers convert SQL into readable, idiomatic LINQ directly in their editor, while making conversion quality and limitations transparent.

Purpose

SQLinq was created for teams that work across SQL and C# every day and need a practical bridge, not a black-box converter. The product goal is to reduce repetitive manual translation while keeping developers in control of the output.

What makes it different

  • Editor-first workflow: conversion happens where code already lives.
  • Explainable pipeline: parse, bind, rewrite, emit.
  • Trust signals: correctness and exact-match metrics are visible.
  • Phase-based delivery: limited scope, measurable quality gates.

Who it is for

SQLinq is designed for .NET developers, technical leads, and teams modernizing data access code. It is especially useful when SQL logic needs to be brought into LINQ-based repositories with less manual effort and fewer regressions.

  • Developers converting legacy SQL to LINQ in active projects.
  • Teams reviewing SQL/LINQ parity as part of pull requests.
  • Architects who need a transparent and extensible conversion path.

How we build

The product is developed in phases so each release is testable and clearly scoped. Phase 1 targets Microsoft SQL Server conversion patterns first, then expands by dialect through provider architecture.

Current direction

  • Improve correctness for common and high-value query patterns.
  • Track failed and partial conversions and tie them to issues.
  • Expose practical performance and reliability indicators.
  • Keep the core reusable across Visual Studio and VS Code hosts.
