Privacy

This page explains what SQLinq website data is used for and how it is handled in the context of a developer-facing product site.

Data we display

The product site primarily displays project information and public repository metrics. Some sections query public GitHub APIs to show stars, forks, issues, pull requests, and commit activity.

  • Public repository metadata from GitHub APIs.
  • Public issue and commit counts for transparency panels.
  • Quality report snapshots generated by project tooling.

Data handling intent

Displayed data is used to improve transparency and release trust. The site does not require user account creation to browse project information pages.

Operational use

  • Show real-time progress for contributors and evaluators.
  • Present conversion quality and failure-tracking indicators.
  • Support release planning with measurable public signals.

Third-party services

When live metrics are enabled, GitHub endpoints are queried for public data. Service behavior and availability may depend on GitHub API limits and network conditions.

Policy updates

As SQLinq evolves, this page may be updated to reflect new data flows, integrations, or telemetry decisions related to product features.

Quick summary

  • Primary sourcePublic GitHub repository data
  • PurposeTransparency and release quality reporting
  • Sign-in requiredNo account needed for page browsing
